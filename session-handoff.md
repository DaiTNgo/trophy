- Storefront home UI spec was captured on 2026-07-05 in `docs/plans/2026-07-05-storefront-home-trophysmack-layout-spec.md`. It adapts TrophySmack's home-page sequence into a Trophy-specific Premium Championship Commerce layout: trust bar, header/search/category strip, full-bleed product hero, proof row, shop-by-product, best sellers, customization feature, occasion paths, real-work gallery, reviews, SEO intro, newsletter, and footer. Search and newsletter inputs should stay compact at 40px height. No implementation files were changed for this spec pass.
- Fixed admin sign-in/session loading on 2026-07-05. Root cause: the admin app had partially migrated to token auth by storing `signIn.username()`'s returned token in `localStorage`, but still used `authClient.getSession()` for session bootstrap and login validation. Better Auth 1.6.23 `/get-session` only reads the signed session cookie, so bearer-only local sessions returned `null`; local cookie auth is also fragile because backend cookies are configured as `sameSite: "none"` + `secure: true`. Added `apps/backend/src/lib/admin-session.ts` to resolve admin sessions from cookie first and `Authorization: Bearer` token second, added `GET /api/admin/me`, updated backend admin guards/account/product-asset paths to use the shared resolver, and updated `apps/admin/src/hooks/use-auth.tsx` to load the current admin user through `/api/admin/me` instead of Better Auth `getSession`. Regression coverage lives in `apps/backend/src/routes/admin/auth-session.test.ts`. Verification passed with targeted backend auth test, backend check/test/build, admin build, and `./init.sh`.
- Updated `AGENTS.md` on 2026-07-04 with the Hono RPC harness rule: new backend route contracts consumed by admin/storefront should export backend route/app types, use typed `hc<AppType>()` clients, avoid new hand-written fetch wrappers unless a blocker is documented, and return explicit typed JSON responses with `c.json(payload, status)` for client-consumed success/error cases. This is a forward-looking harness rule; existing hand-written clients were not migrated in this pass.
- Fixed admin product detail publish status display on 2026-07-04. Root cause: `POST /api/admin/products/:id/publish` correctly updates/returns lowercase `status: "published"`, but `apps/admin/src/lib/products-client.ts` typed API status as PascalCase and only treated `"Published"` as published, so `mutate()` refreshed the product and mapped it back to UI `Draft`. Added status normalization in the admin mapper and regression coverage in `apps/admin/src/lib/products-client.test.ts`. Verified with `pnpm --filter admin test -- products-client.test.ts`, `pnpm --filter admin build`, and `./init.sh`.
- Root harness was updated on 2026-07-04 to make backend API contract tests part of Definition of Done. `AGENTS.md` now requires every new/changed backend route to have public route-surface API contract coverage for admin/storefront consumers, important failure-mode tests, service/helper unit tests for non-trivial business logic, and recorded risk when tests are intentionally omitted. `init.sh` now runs backend check, backend test, and backend build before admin/storefront verification. Verification passed with `pnpm --filter backend check`, `pnpm --filter backend test`, `pnpm --filter backend build`, and `./init.sh`.
- Storefront SSR/hydration check on 2026-07-04 found no hydration warnings in browser automation for `/`, `/cart`, `/checkout`, `/customize`, or `/products` after the fix. The reproduced failure was SSR loader/API related: `/products` returned 500 because storefront server loaders defaulted to `http://127.0.0.1:8787`, but the backend dev server was reachable at `http://localhost:8787` and not `127.0.0.1:8787`. Updated the fallback URL in `apps/storefront/app/lib/api.ts`, `apps/storefront/app/components/CupCustomizer.tsx`, `apps/storefront/app/routes/customize.tsx`, and `apps/storefront/app/routes/customize.$templateId.tsx`. Verified with `pnpm --filter router-cf typecheck`, preview build, curl `/products` returning 200, Playwright reload checks with no console errors, and `./init.sh`.
