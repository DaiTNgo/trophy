# Admin Categories Page Proposal

## Why

The admin needs a Medusa-like categories page so operators can manage catalog taxonomy with a clear tree structure. Categories solve navigation and classification, while collections solve curation, so both capabilities need separate workflows.

## What Changes

- Add a Medusa-like categories management page to the admin spec.
- Define hierarchical category behaviors and tree constraints.
- Define mock-first contracts for listing, creating, moving, and editing categories.

## Impact

- Admin planning can implement category tree management with stable expectations.
- Backend planning can prepare a category hierarchy model and mutation rules.
- Frontend teams can ship mock-first tree interactions before live APIs exist.
